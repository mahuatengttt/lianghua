## 一、沟通风格观察

### 1.1 高效简洁
马总说话极其精简——大部分指令不超过15个字：
- "创建文件，把你今天的开发心得写进去"
- "回测结果给我看下"
- "好"
- "github…f5tm"（直接把token发过来，不讲废话）

这意味着回复也要对等——结论先行，细节后补，不要铺垫。

### 1.2 技术判断力强
- 一眼看出德明利的股票代码错了（我说002290，他纠正为001309）
- 意识到需要对比多策略而不是只看一组参数
- 对交易明细感兴趣，说明他要的不是"收益率数字"，而是策略具体怎么操作的

### 1.3 不纠结过程，只看结果
- 回测跑出几组数据后，他没让我分析方差、分布等细节，直接看收益率、回撤、夏普、交易笔数
- 遇到技术问题（GitHub token权限、浏览器起不来）时，他直接甩新token或问"怎么搞"，不自己折腾

## 二、产品需求洞察

### 2.1 真正的需求
马总关注的不是"系统有多酷"，而是：
1. **策略在真实数据上能不能赚钱**（德明利回测683%）
2. **有没有bug**（旧版GridMA状态bug被他抓到了）
3. **能不能用**（数据源拉不通→换雅虎，token不通→他直接给新token）
4. **能不能落地**（最终推到GitHub，为团队协作做准备）

### 2.2 部署环境限制
- Python 3.6（无法用AKShare等现代包）
- 服务器外网受限（部分域名不通，雅虎有时也慢）
- 没有浏览器图形界面（浏览器启动失败）
- 这种环境下越简单越可靠

### 2.3 信任建立在结果上
一开始马总还是试探性的（"请提供系统技术说明"），但当看到德明利数据拉通、回测跑出真实结果、bug逐个被修复、收益数据具体可查之后，指令变得更直接、更信任。

## 三、有价值的开发教训

### 3.1 先跑通，再优化
第一天花了很多时间搭框架、写文档，结果数据源都拉不通。真正开始跑数据才暴露所有问题：
- AKShare不支持Python 3.6
- analyzer统计有bug
- 策略状态管理有问题
- 股票代码搞错

**教训：量化系统的价值从第一行真实数据开始，不是从第一个类定义开始。**

### 3.2 代码里的bug只能在回测中发现
GridMA策略的逻辑从代码上看"似乎没问题"，但回测日志一打出来就露馅了——每天都在重新建仓。如果没有实际跑回测，这个bug可能在代码审查时完全被忽略。

**教训：量化策略必须实际回测，不能只看代码。**

### 3.3 回测收益高不等于好策略
旧版GridMA因为状态管理bug显示+682%（虚假收益），修复后+484%。趋势网格在德明利跑赢买入持有（683% vs 511%），但回撤高达50%。DualMA(5/20)收益低一些（450%）但回撤只有29%。

**教训：多指标综合评价比只看收益率重要。**

### 3.4 和真人沟通的经验
- **纠错要干脆**——马总说代码错了时直接说"弄错了吧"，我查了确认代码是002290不是001309，直接认错+补救，不辩解
- **不懂就做选择**——Token权限问题花了10多分钟尝试各种方法，最后SSH一次成功。马总给方案时给选项，让他做选择题而不是填空题
- **少说多做**——马总习惯给指令而不是讨论，收到指令立刻执行、输出结果，比问"你想怎么处理"好得多

## 四、量子系统后续规划建议

基于这次对话，我认为马总希望这个系统最终：

1. **实盘可用** — 不是学术研究项目，是要真金白银下单的
2. **回测靠谱** — 策略状态管理必须精确，不能有虚假收益
3. **多标的覆盖** — 不是只在德明利上跑得通，要在茅台、宁德、工行都验证过
4. **参数可调** — 研究员（而不是程序员）也能改配置跑回测
5. **团队协作** — 推到GitHub是为团队铺路

### 接下来的优化方向（按优先级）:
1. ✅ 修复夏普异常值（空仓期除零问题）
2. ✅ 多策略组合仓位管理验证
3. ⬜ 做空策略（A股融券/ETF期权）
4. ⬜ 实盘Broker对接（XTP/QMT）
5. ⬜ HTML回测报告生成
6. ⬜ 分钟级回测支持

## 五、和"马化腾"这个身份合作的有趣观察

虽然马总是中国互联网的传奇人物，但沟通中完全没有任何"大佬架子"。直接用Telegram，发消息很随意，遇到技术问题直接说"怎么搞，我不会"，和普通开发者一样。这种务实风格让人很舒服，沟通效率极高。

但反过来——**他对代码质量和结果的要求也极高**。一个股票代码错了、策略有bug、回测收益有水分，他都能感觉到。这不是技术深度的体现（他可能不自己写代码），而是**对"靠谱"二字的极度敏感**——在做投资的人眼里，数据和逻辑上的任何一个"不靠谱"都是致命伤。

**这提醒我：给做量化的人干活，准确性比速度重要，诚实比漂亮重要。**
